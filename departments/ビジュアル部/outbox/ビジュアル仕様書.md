## 2. 各要素の描画方針

### 2-1. 宇宙背景（`background_visual.js`）

**形状・色・サイズ**
- 背景全体：黒（`#000000`）で 400×600 px を塗りつぶし
- 星：直径 1〜3 px の白〜薄青の円（`arc`）
  - 大（直径 2〜3 px）：`#FFFFFF`、少数（全体の 20%）
  - 中（直径 1〜2 px）：`#CCDEFF`、中程度（全体の 30%）
  - 小（直径 1 px）：`#8AA8CC`、多数（全体の 50%）
- 星の数：80〜120 個（初期化時に乱数で配置）

**使用する Canvas API**
- `fillRect`：背景塗りつぶし
- `arc` + `fill`：各星の描画

**アニメーション**
- 各フレーム、stars 配列の y 座標を `speed` px ずつ増加（下方向スクロール）
- y が 600 を超えた星は y = 0（または負値）にリセット、x は乱数で再設定
- スクロール速度：大星 1.5 px/frame、中星 1.0 px/frame、小星 0.5 px/frame（視差効果）

---

### 2-2. 自機・スペースシャトル（`ship_visual.js`）

**形状・色・サイズ**
- デフォルトサイズ：幅 40 px × 高さ 50 px（x, y は中心座標）
- 胴体：白（`#FFFFFF`）の角丸矩形（幅 16 px × 高さ 38 px）
- 翼（左右）：薄グレー（`#CCCCCC`）の三角形
  - 左翼：胴体左端から外側に 14 px 広がり、前端に向かって収束
  - 右翼：左右対称
- ノーズコーン（先端）：白（`#FFFFFF`）の三角形（高さ 12 px）
- エンジン排気口：濃グレー（`#888888`）の小矩形（幅 10 px × 高さ 6 px）を胴体後端に配置
- エンジン炎（スラスター）：オレンジ〜黄色のグラデーション三角形、`createLinearGradient` で表現
  - 先端色：`#FFEE44`、後端色：`#FF6600`（透明度あり）
  - 高さ：8〜12 px（フレームごとに微変動で炎感を出す）

**使用する Canvas API**
- `beginPath` / `lineTo` / `fill`：翼・ノーズコーン・炎の多角形
- `fillRect` / `roundRect`（または `arc` で代替）：胴体
- `createLinearGradient`：エンジン炎
- `save` / `restore`：座標変換の隔離

**アニメーション**
- エンジン炎の高さを `Math.sin(frame * 0.3) * 2 + 10` px 程度で揺らす（呼び出し元から `frame` を渡す）

---

### 2-3. 敵A：UFO型（`enemy_visual.js`）

**形状・色・サイズ**
- デフォルトサイズ：幅 48 px × 高さ 24 px
- 下半球（ソーサー本体）：濃グレー〜シルバー（`#555577`）の楕円（幅 48 px、高さ 14 px）
- 上ドーム（キャノピー）：半透明の薄緑（`rgba(0, 255, 120, 0.6)`）の楕円（幅 24 px、高さ 14 px）
- アクセントライト：ソーサー円周上に小円（直径 4 px）を 5〜6 個等間隔に配置
  - 色：交互に `#FF4444`（赤）/ `#FFFF44`（黄）で点滅感を演出
- ハイライト：ソーサー上面に `rgba(255,255,255,0.2)` の細長い楕円

**使用する Canvas API**
- `ellipse`：本体・ドーム・ハイライト
- `arc`：アクセントライト
- `fill`

**アニメーション**
- アクセントライトの色を `frame % 2` で切り替えて点滅させる（frame を引数で受け取る）

---

### 2-4. 敵B：隕石型（`enemy_visual.js`）

**形状・色・サイズ**
- デフォルトサイズ：幅 36 px × 高さ 36 px
- 本体：不規則な多角形（8〜12頂点の凸多角形）
  - 基本色：濃茶（`#6B4226`）、シャドウ側：`#3D1F0A`
- クレーター：本体上に小楕円 3〜4 個（`rgba(0,0,0,0.4)`）
- ハイライト：左上方向に `rgba(255,200,150,0.3)` の小楕円

**使用する Canvas API**
- `beginPath` / `lineTo` / `fill`：不規則本体
- `ellipse`：クレーター・ハイライト
- `save` / `rotate` / `restore`：回転（frame を使って緩やかに回転）

**アニメーション**
- `ctx.rotate(frame * 0.02)` で緩やかに回転（frame を引数で受け取る）

---

### 2-5. プレイヤー弾（`objects_visual.js`）

**形状・色・サイズ**
- 幅 4 px × 高さ 14 px の縦長矩形
- メイン色：明るい黄色（`#FFEE00`）
- 発光コア：中央に幅 2 px × 高さ 10 px の白（`#FFFFFF`）を重ねてレーザー感を演出
- グロー効果：`shadowColor = '#FFEE00'`、`shadowBlur = 8` で外縁を光らせる

**使用する Canvas API**
- `fillRect`：メイン弾体・コア
- `shadowColor` / `shadowBlur`：グロー効果

**アニメーション**
- なし（位置更新はゲームロジック側が担当）

---

### 2-6. ゴールオブジェクト（`objects_visual.js`）

**形状・色・サイズ**
- 幅 80 px × 高さ 40 px（x, y は中心座標）
- 外枠：明るいシアン（`#00FFFF`）の矩形ボーダー（線幅 3 px）
- 内部：薄シアン（`rgba(0,255,255,0.15)`）で塗りつぶし
- テキスト：「GOAL」の文字を中央に白（`#FFFFFF`）、フォント `bold 18px monospace`
- 光の波紋：外枠から広がる同心矩形を frame で縮小→拡大ループさせる
  - 色：`rgba(0,255,255,0.3 * (1 - t))` ← t は 0〜1 の波紋進捗

**使用する Canvas API**
- `strokeRect` / `fillRect`：外枠・内部
- `fillText`：「GOAL」テキスト
- `strokeRect`：波紋エフェクト

**アニメーション**
- 波紋エフェクトあり（frame を引数で受け取る）

---

### 2-7. 爆発エフェクト（`effects_visual.js`）

**形状・色・サイズ**
- 総フレーム数：20 frame（`maxFrame = 20`）
- フェーズ1（frame 0〜7）：輝く核
  - 中央に白（`#FFFFFF`）の明るい円、半径 `frame * 2 + 4` px
  - 外縁に `#FFEE00`（黄）の輪
- フェーズ2（frame 8〜14）：炎の拡散
  - オレンジ（`#FF6600`）〜赤（`#FF2200`）のランダムな三角形パーティクル 6〜8 個
  - 各パーティクルは中心から外方向に `frame * 3` px ほど離れる
- フェーズ3（frame 15〜19）：フェードアウト
  - 全体の `globalAlpha` を `(maxFrame - frame) / maxFrame` で徐々に透明に

**使用する Canvas API**
- `arc` + `fill`：核・輪
- `beginPath` / `lineTo` / `fill`：炎パーティクル三角形
- `globalAlpha`：フェードアウト
- `save` / `restore`：globalAlpha の隔離

**アニメーション**
- 呼び出し元から `frame`（0〜maxFrame-1）と `maxFrame` を渡す
- frame が maxFrame を超えたら描画しない（呼び出し元が管理）

## 5. 実装優先順位

| 優先度 | ファイル | 理由 |
|---|---|---|
| 1 | `background_visual.js` | ゲームループの基盤。毎フレーム呼ばれる最重要描画 |
| 2 | `ship_visual.js` | プレイヤーが常時画面に表示されるため視認性確認に必須 |
| 3 | `objects_visual.js`（弾のみ） | 攻撃感・フィードバックに直結 |
| 4 | `enemy_visual.js` | ゲームの核心。UFO → 隕石の順に実装 |
| 5 | `effects_visual.js` | 爆発エフェクトは爆快感を担うが、ロジックに非依存 |
| 6 | `objects_visual.js`（ゴール） | 30秒後に出現するため後回しにしても開発初期に支障なし |

## 6. GDD 連携時の改訂ポイント

GDD が作成された場合、以下の点を確認・更新すること：

- 敵の種類・数（敵C以降の追加の可能性）
- 各キャラクターの当たり判定サイズ（描画サイズと合わせる必要あり）
- スコア表示・UI テキストの描画要件（ゲーム実装部との分担確認）
- ゴールオブジェクトの演出詳細（演出時間等）
- 特殊エフェクト（被弾、シールド等）の有無

## 7. ゲーム実装部との連携インターフェース

ゲーム実装部が各 draw 関数を呼び出す際の想定コードイメージ：

```js
// ゲームループ内（毎フレーム）
import { drawBackground, updateStars } from './visual/background_visual.js';
import { drawShip } from './visual/ship_visual.js';
import { drawEnemy } from './visual/enemy_visual.js';
import { drawBullet, drawGoal } from './visual/objects_visual.js';
import { drawExplosion } from './visual/effects_visual.js';

function gameLoop(frame) {
  // 背景
  updateStars(stars, CANVAS_H);
  drawBackground(ctx, stars);

  // 自機
  drawShip(ctx, player.x, player.y, 40, 50, frame);

  // 敵
  enemies.forEach(e => drawEnemy(ctx, e.type, e.x, e.y, e.w, e.h, frame));

  // 弾
  bullets.forEach(b => drawBullet(ctx, b.x, b.y));

  // 爆発
  explosions.forEach(ex => {
    drawExplosion(ctx, ex.x, ex.y, ex.frame, 20);
    ex.frame++;
  });

  // ゴール
  if (goalVisible) drawGoal(ctx, goal.x, goal.y, frame);

  requestAnimationFrame(() => gameLoop(frame + 1));
}
```

ファイルは `visual/` サブディレクトリにまとめることを推奨するが、最終的なディレクトリ構成はゲーム実装部と調整すること。
