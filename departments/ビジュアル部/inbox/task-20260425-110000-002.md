## エクスポート形式について

仕様書には `export function` と書かれているが、**ゲーム実装部側はモジュールシステムを使わず `<script>` タグで読み込む方針**（実装設計書より）。

そこで以下の両対応で実装する：

```javascript
// ファイル末尾で
if (typeof module !== "undefined" && module.exports) {
  module.exports = { drawShip };
}
if (typeof window !== "undefined") {
  window.ShipVisual = { draw: drawShip };
}
```

つまり **`window.ShipVisual.draw(ctx, x, y, invincible)` のような、グローバル名前空間経由で呼べる形**にする。名前空間：

| ファイル | グローバル |
|---|---|
| background_visual.js | `window.BackgroundVisual = { init, update, draw }` |
| ship_visual.js | `window.ShipVisual = { draw }` |
| enemy_visual.js | `window.EnemyVisual = { draw }` |
| objects_visual.js | `window.ObjectsVisual = { drawBullet, drawGoal }` |
| effects_visual.js | `window.EffectsVisual = { drawExplosion }` |

## 座標系の注意

**仕様書は中心座標基準、実装設計書は左上基準**と食い違っている。

→ **draw 関数は中心座標 (x, y) を受け取る**（仕様書優先）。
→ ゲーム実装部との統合時に「自機 x, y は中心座標として渡す」ことを実装部に通達済み（実装設計書6節）。
→ 報告書にこの点を明記しておく。
