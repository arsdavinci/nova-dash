# ビジュアル部 — シューティングゲーム

## 役割

宇宙を舞台にしたシューティングゲームのグラフィックを、HTML5 Canvas 上で描画する JavaScript コードとして制作する部署。画像ファイルは使わず、Canvas API（`fillRect`, `arc`, `beginPath` 等）または SVG インライン形式で、すべてのビジュアル要素をコードで表現する。

## 守備範囲

- 自機（スペースシャトル）の Canvas 描画コード
- 敵キャラクターの Canvas 描画コード（複数種類あれば複数）
- 宇宙背景の Canvas 描画コード（星空、星のスクロール等）
- 弾（プレイヤー弾）の描画コード
- 爆発・消滅エフェクトの描画コード
- ゴールオブジェクトの描画コード（30秒後に出現するもの）

## 主要成果物

- `ship_visual.js`：自機の描画関数
- `enemy_visual.js`：敵の描画関数
- `background_visual.js`：背景（星空スクロール）の描画関数
- `effects_visual.js`：爆発・エフェクトの描画関数
- `objects_visual.js`：弾・ゴール等その他オブジェクトの描画関数

## 出力ルール

- 成果物の保存先：`outbox/` 配下
- 報告書ファイル名：`report-YYYYMMDD-HHmmss.md`
- 各ファイルは `export function draw<Object>(ctx, x, y, ...)` の形式で関数を公開
- Canvas context (`ctx`) を受け取って描画する純粋関数として実装
- 完了後 "done" で締める

## 参照リソース

- プロジェクトコンテキスト：`../../CLAUDE.md`
- ゲームデザイン部の仕様書：`../ゲームデザイン部/outbox/GDD.md`（ビジュアル仕様を参照）

## 他部署との連携

### 受け取るもの
- ゲームデザイン部 から：GDD.md（自機・敵・背景のビジュアル方向性）

### 渡すもの
- ゲーム実装部 へ：各描画モジュール（統合して使用）

## このプロジェクト固有の考慮事項

- テーマ：宇宙空間。黒背景に星が流れる、SF的な雰囲気
- 自機：スペースシャトルらしいシルエット（白・グレー系）
- 敵：宇宙的な外観（UFO、隕石、エイリアン機等、GDD指定に従う）
- 画像ファイル（PNG/JPG）は使用不可。Canvas API のみで描画
- シンプルでも良い。ゲームとして識別できる視認性を優先

## 禁止事項

- 外部画像ファイルを参照しない（Canvas API のみ使用）
- ゲームロジックを描画コードに混ぜない（描画のみに専念）
- `document.getElementById` 等 DOM 操作を直接行わない（ctx は引数で受け取る）

## ノウハウ蓄積

Canvas で宇宙系グラフィックを描くパターン（星のスクロール手法、シャトルの描き方等）は `knowledge/` に追記。
