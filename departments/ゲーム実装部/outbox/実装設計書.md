## 1. ファイル構成と各ファイルの責務

```
shooting-game/
├── index.html          - エントリポイント。Canvas要素の配置、JSファイルの読み込み順序を管理
├── game.js             - ゲームループ（requestAnimationFrame）、状態管理（title/playing/clear/gameover）、
│                         全オブジェクトの統括（Player, Enemy[], Bullet[]）
├── player.js           - Playerクラス。自機の座標・速度・HP・描画・移動処理・射撃クールダウン管理
├── enemy.js            - Enemyクラス。敵の座標・移動パターン・HP・消滅判定・描画
├── bullet.js           - Bulletクラス。弾の座標・移動速度・画面外判定・描画
├── background.js       - Backgroundクラス。星レイヤー（近景/遠景）の座標配列管理、ループスクロール描画
├── collision.js        - AABB衝突判定関数を提供するモジュール。クラスに依存しない純粋関数群
└── （統合時追加）
    ├── ship_visual.js  - ビジュアル部が提供する自機描画モジュール
    ├── enemy_visual.js - ビジュアル部が提供する敵描画モジュール
    └── sound.js        - サウンド部が提供するWeb Audio API BGM・SEモジュール
```

### 読み込み順序（index.html の script タグ順）

```
collision.js → background.js → bullet.js → enemy.js → player.js → game.js
```

依存関係上、collision.js（純粋関数）を最初に読み込み、game.js を最後にすることでクラス未定義エラーを防ぐ。統合モジュール（ship_visual.js, enemy_visual.js, sound.js）は player.js / enemy.js より前に読み込む。

## 2. クラス設計

### 2-1. Player クラス（player.js）

```
プロパティ：
  x, y            - 自機の中心座標（初期値：Canvas中央下部 200, 520）
  width, height   - ヒットボックスサイズ（32, 32）
  speed           - 移動速度（px/frame）: 4
  hp              - ヒットポイント（初期値：3）
  shootCooldown   - 射撃クールダウンカウンタ（0になったら発射可能）
  shootInterval   - 射撃間隔（フレーム数）: 15（＝60fps時0.25秒）
  invincible      - 被弾後無敵フレームカウンタ
  bullets         - 自機が発射した Bullet[] （game.js で一元管理でも可）

メソッド：
  update(keys)    - キー入力に基づいて座標更新、Canvasクランプ、クールダウン減算
  shoot()         - cooldownが0なら Bullet を生成して返す（game.js 側で配列に追加）
  takeDamage()    - hp減算、無敵フレームセット
  draw(ctx)       - Canvas に自機を描画（ship_visual.js があれば委譲）
  getBounds()     - { x, y, width, height } を返す（衝突判定用）
```

### 2-2. Enemy クラス（enemy.js）

```
プロパティ：
  x, y            - 座標（画面上端 y=-32 から出現）
  width, height   - ヒットボックスサイズ（32, 32）
  speed           - 降下速度（px/frame）: 2〜3（出現タイプにより可変）
  hp              - ヒットポイント（初期値：1〜3）
  type            - 敵タイプ識別子（"straight", "zigzag", "boss"等）
  active          - 生存フラグ（false で削除対象）

メソッド：
  update()        - 移動パターン実行、画面下端で active=false
  takeDamage(dmg) - hp 減算、0以下で active=false、スコア加算イベント発行
  draw(ctx)       - 描画（enemy_visual.js があれば委譲）
  getBounds()     - { x, y, width, height } 返す
```

### 2-3. Bullet クラス（bullet.js）

```
プロパティ：
  x, y            - 座標
  width, height   - ヒットボックス（4, 12）
  speedY          - 飛翔速度（-10 px/frame：上方向）
  owner           - "player" or "enemy"（将来的に敵弾対応する際の識別）
  active          - 生存フラグ

メソッド：
  update()        - y += speedY、画面外（y < -16）で active=false
  draw(ctx)       - 矩形またはビジュアルモジュールで描画
  getBounds()     - { x, y, width, height } 返す
```

### 2-4. Background クラス（background.js）

```
プロパティ：
  stars           - 星オブジェクト配列 { x, y, size, speed, brightness }
  STAR_COUNT      - 星の総数（例：80）

メソッド：
  constructor()   - ランダム座標で stars[] を初期化（近景30個, 遠景50個の2層）
  update()        - 各星の y を speed 分加算、600以上になったら y=0 にリセット
  draw(ctx)       - ctx.fillRect で白/グレーの点を描画
```

### 2-5. collision.js（純粋関数モジュール）

```javascript
// AABB（軸平行境界ボックス）衝突判定
function isColliding(a, b) {
  // a, b: { x, y, width, height }（左上座標基準）
  return (
    a.x < b.x + b.width &&
    a.x + a.width > b.x &&
    a.y < b.y + b.height &&
    a.y + a.height > b.y
  );
}

// 複数 vs 複数のバッチ判定（game.js から呼び出す）

```

## 3. ゲームループ設計

### 状態定義

```
STATE = {
  TITLE:    "title",
  PLAYING:  "playing",
  CLEAR:    "clear",
  GAMEOVER: "gameover"
}
```

### メインループ（game.js）

```
gameLoop(timestamp) {
  deltaTime = timestamp - lastTimestamp
  lastTimestamp = timestamp

  switch(currentState) {
    case TITLE:
      drawTitleScreen()
      if (Enterキー押下) → currentState = PLAYING, initGame()
      break

    case PLAYING:
      // --- update phase ---
      background.update()
      player.update(keys)
      handleShooting()          // Aキー → player.shoot() → bullets に追加
      updateBullets()           // 全弾の update()、active=false を除去
      updateEnemies()           // 全敵の update()、active=false を除去
      spawnEnemies()            // タイマーに基づき敵を生成
      checkCollisions()         // collision.js を使って判定、スコア更新
      updateTimer()             // 経過時間を加算
      checkGoalCondition()      // 30秒経過 → ゴール出現、触れたら CLEAR
      if (player.hp <= 0) → currentState = GAMEOVER

      // --- draw phase ---
      ctx.clearRect(0, 0, 400, 600)
      background.draw(ctx)
      drawBullets()
      drawEnemies()
      player.draw(ctx)
      drawGoal()                // 出現後のみ
      drawHUD()                 // スコア、HP、残り時間

      break

    case CLEAR:
      drawClearScreen()
      if (Enterキー) → currentState = TITLE

    case GAMEOVER:
      drawGameOverScreen()
      if (Enterキー) → currentState = TITLE
  }

  requestAnimationFrame(gameLoop)
}
```

### タイマー設計

```
playTime       - PLAYING 開始からの累積秒数（deltaTime を 1000 で割って加算）
GOAL_TIME      - 30（秒）
goalSpawned    - ゴール出現フラグ（boolean）
goalY          - ゴールオブジェクトの y 座標（上から降下）
```

## 4. 入力管理

`game.js` 内でグローバルに `keys` オブジェクトを管理する。

```javascript
const keys = {};

window.addEventListener("keydown", (e) => {
  keys[e.key] = true;
  // Enterキーによる状態遷移（1フレーム内で1回だけ処理するため別フラグも検討）
  if (e.key === "Enter") handleEnterPress();
});

window.addEventListener("keyup", (e) => {
  keys[e.key] = false;
});
```

移動判定（player.update 内）:

```javascript
if (keys["ArrowUp"])    this.y -= this.speed;
if (keys["ArrowDown"])  this.y += this.speed;
if (keys["ArrowLeft"])  this.x -= this.speed;
if (keys["ArrowRight"]) this.x += this.speed;
```

射撃判定（handleShooting 内）:

```javascript
if (keys["a"] || keys["A"]) {
  const b = player.shoot();  // クールダウンが0の時のみ Bullet を返す
  if (b) bullets.push(b);
}
```

### 注意事項

- `e.preventDefault()` はゲーム中（PLAYING 状態）の矢印キーに対してのみ呼ぶ（ページスクロール防止）
- 長押し連射は shootInterval で制御。`keys["a"]` のフラグ on/off ではなくクールダウンで管理

## 6. ビジュアル部・サウンド部の統合方針

### 想定モジュール構成

ビジュアル部・サウンド部は以下の形式でモジュールを提供してもらう想定：

```javascript
// ship_visual.js（例）
const ShipVisual = {
  draw(ctx, x, y, invincible) {
    // ctx に自機グラフィックを描画
  }
};

// enemy_visual.js（例）
const EnemyVisual = {
  draw(ctx, x, y, type) {
    // 敵タイプに応じた描画
  }
};

// sound.js（例）
const Sound = {
  init() { /* AudioContext 初期化 */ },
  playBGM() { ... },
  stopBGM() { ... },
  playSE(name) { /* "shoot", "explosion", "hit", "clear" 等 */ }
};
```

### 統合方法（フォールバック設計）

各描画クラスの draw メソッドに条件分岐を設ける。**ビジュアルモジュールがない状態でも動作する** ことを保証する。

```javascript
// player.js の draw
draw(ctx) {
  if (typeof ShipVisual !== "undefined") {
    ShipVisual.draw(ctx, this.x, this.y, this.invincible > 0);
  } else {
    // フォールバック：プレースホルダ矩形描画
    ctx.fillStyle = this.invincible > 0 ? "rgba(100,200,255,0.5)" : "#00BFFF";
    ctx.fillRect(this.x - this.width/2, this.y - this.height/2, this.width, this.height);
  }
}
```

同様に enemy.js、bullet.js にもフォールバック描画を実装する。

### サウンド統合タイミング

- `initGame()` 内で `Sound.init()` と `Sound.playBGM()` を呼ぶ
- 弾発射時：`Sound.playSE("shoot")`
- 敵消滅時：`Sound.playSE("explosion")`
- 被弾時：`Sound.playSE("hit")`
- クリア時：`Sound.stopBGM()`、`Sound.playSE("clear")`
- ゲームオーバー時：`Sound.stopBGM()`

サウンドモジュールが未提供の場合は `typeof Sound !== "undefined"` で分岐し、サウンドなしで動作させる。

### ビジュアル部・サウンド部への依頼事項

- モジュール名（グローバル変数名）を事前に合意する
- フォールバック描画との切り替えをスムーズにするため、draw/playSE の引数仕様を共有する
- Canvas座標系：自機 x,y は **中心座標** として渡すことを明示する

## 7. リスクと対策

### リスク 1：requestAnimationFrame の deltaTime ばらつき

**問題**：タブがバックグラウンドになると deltaTime が急増し、オブジェクトが瞬間移動する。

**対策**：
```javascript
const MAX_DELTA = 50; // 50ms（20fps相当）を上限にクランプ
deltaTime = Math.min(timestamp - lastTimestamp, MAX_DELTA);
```

ただし、このゲームはフレームレート固定（60fps想定）で speed を px/frame で管理しているため、deltaTime をそのまま使わず「1フレーム＝1/60秒相当で固定」とする設計が実装シンプル。タイマー（経過秒数）のみ deltaTime を使う。

### リスク 2：衝突判定のコスト

**問題**：敵・弾の数が増えると O(n×m) の判定がボトルネックになる可能性。

**対策**：
- 敵は同時画面内最大20体、弾は最大30発程度に上限を設ける（スポーン制御で自然に制限される）
- 範囲内なら全数探索で60fps維持は可能（最大600回の矩形判定は十分高速）
- active=false のオブジェクトは配列から即削除（`filter`）してイテレーション数を減らす

### リスク 3：keydown の連射判定

**問題**：OS のキーリピートが介入し、keydown が連続発火して意図しない連射になる。

**対策**：射撃はキーの on/off ではなく `shootCooldown` カウンタで制御。`keys["a"]` が true の間は毎フレーム shoot() を呼ぶが、cooldown が 0 の時のみ Bullet を生成する。

### リスク 4：Web Audio API のオートプレイ制限

**問題**：ブラウザのオートプレイポリシーにより、ユーザー操作前に AudioContext を開始できない。

**対策**：`Sound.init()` をタイトル画面でのキー入力（Enter押下）のタイミングで呼ぶ。AudioContext の作成はユーザー操作イベントハンドラ内で行う。

### リスク 5：Canvas のぼやけ（高DPI環境）

**問題**：Retina/高DPI ディスプレイで Canvas がぼやけることがある。

**対策**：
```javascript
const dpr = window.devicePixelRatio || 1;
canvas.width = 400 * dpr;
canvas.height = 600 * dpr;
canvas.style.width = "400px";
canvas.style.height = "600px";
ctx.scale(dpr, dpr);
```

ただしビジュアル品質よりシンプルさを優先するなら、通常の `400×600` 固定でも問題ない。

### リスク 6：ビジュアル部・サウンド部との統合タイミングのズレ

**問題**：実装部がゲームを完成させた後でビジュアル/サウンドを差し込む場合、インターフェース不一致が生じやすい。

**対策**：早期にモジュール名・関数名・引数仕様を文書化（本設計書の第6節）して共有。実装部はフォールバック描画でフェーズ1〜5を完成させてから統合する。

## 付録：Canvas 座標系と定数一覧

```
CANVAS_WIDTH   = 400
CANVAS_HEIGHT  = 600
FPS_TARGET     = 60
PLAYER_SPEED   = 4          // px/frame
BULLET_SPEED   = -10        // px/frame（上方向）
SHOOT_INTERVAL = 15         // frames（= 0.25秒 @ 60fps）
ENEMY_SPEED_BASE = 2        // px/frame
GOAL_TIME      = 30         // seconds
STAR_COUNT     = 80         // 背景の星総数
MAX_ENEMIES    = 20         // 同時存在上限
MAX_BULLETS    = 30         // 同時存在上限
```

自機・敵の x, y は **矩形の左上座標** を基準とする（getBounds() も左上基準）。ただし draw() 内部での描画位置計算に注意（ヒットボックス中心と描画中心を合わせる）。
