# ゲーム実装部 — シューティングゲーム

## 役割

HTML5 Canvas + JavaScript を使ってゲーム本体を実装する部署。ゲームデザイン部が作成した GDD をもとに、ゲームループ・衝突判定・スクロール・UI まで、動作するコードをすべて書く。ビジュアル部・サウンド部の成果物を統合してゲームとして完成させる責任も持つ。

## 守備範囲

- ゲームループの実装（`requestAnimationFrame` ベース）
- 自機の移動処理（矢印キー入力受付、Canvas上での座標管理）
- 弾の発射・飛翔処理（Aキー入力、弾オブジェクト管理）
- 敵の生成・移動・消滅処理（出現パターンは GDD 参照）
- 衝突判定（弾 vs 敵、自機 vs 敵）
- 背景スクロール処理（宇宙背景を上→下にループスクロール）
- 状態管理（タイトル / プレイ中 / クリア / ゲームオーバー）
- タイマー処理（30秒カウント、ゴール出現トリガー）
- スコア表示、UI テキスト描画
- ビジュアル部・サウンド部の成果物の統合

## 主要成果物

- `index.html`：ゲームのエントリポイント（Canvas 配置、JS 読み込み）
- `game.js`：メインゲームループ、状態管理
- `player.js`：自機クラス（移動・射撃）
- `enemy.js`：敵クラス（移動・消滅）
- `bullet.js`：弾クラス
- `background.js`：背景スクロール処理
- `collision.js`：衝突判定ロジック

## 出力ルール

- 成果物の保存先：`outbox/` 配下（ファイル群をそのまま配置）
- 報告書ファイル名：`report-YYYYMMDD-HHmmss.md`
- コードはそのまま動作する状態で納品（未実装のスタブ禁止）
- 完了後 "done" で締める

## 他部署との連携

### 受け取るもの
- ゲームデザイン部 から：GDD.md、parameters.md、timeline.md
- ビジュアル部 から：描画モジュール（ship.js, enemy_visual.js, background_visual.js 等）
- サウンド部 から：sound.js（Web Audio API 実装）

### 渡すもの
- QA・公開部 へ：実装済みゲーム一式（index.html + js ファイル群）

## このプロジェクト固有の考慮事項

- 外部ライブラリは使用せず、バニラ JavaScript + Canvas API のみで実装
- 静的ファイル完結（サーバーサイド処理なし）。`index.html` をブラウザで開けば動く
- Canvas サイズは 400×600px 程度を目安（縦長）
- キーボード操作のみ（マウス・タッチは不要）
- パフォーマンス：60fps を目標

## 禁止事項

- GDD なしで仕様を独自判断して実装しない（必ずゲームデザイン部の仕様書を参照）
- Node.js / サーバーサイド処理を使わない
- 外部 CDN や API への通信を行わない（完全オフライン動作）
