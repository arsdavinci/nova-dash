## 背景

君（ゲーム実装部）は既に `outbox/` に以下を実装済み：

```
outbox/
  index.html
  game.js
  player.js
  enemy.js
  bullet.js
  background.js
  collision.js
  実装設計書.md
```

一方、別部署が以下を完成させてる：

- **ビジュアル部** (`../ビジュアル部/outbox/`):
  - `ship_visual.js` → `window.ShipVisual.draw(ctx, x, y, width, height, frame)` ※**中心座標**
  - `enemy_visual.js` → `window.EnemyVisual.draw(ctx, type, x, y, width, height, frame)` ※**中心座標** / type: `'ufo' | 'meteor'`
  - `background_visual.js` → `window.BackgroundVisual.{ initStars(count, W, H), updateStars(stars, dt), drawStars(ctx, stars, W, H) }` ※関数名は実ファイルを読んで確認すること
  - `objects_visual.js` → `window.ObjectsVisual.{ drawBullet(ctx, x, y), drawGoal(ctx, x, y, frame) }` ※**中心座標**
  - `effects_visual.js` → 爆発などのエフェクト（実ファイル読んで API 確認）
- **サウンド部** (`../サウンド部/outbox/`):
  - `sound.js` → `class SoundManager` / `.resume()` / `.playBGM()` / `.stopBGM()` / `.playSE('shoot'|'explosion'|'playerHit'|'gameover'|'clear')` / `.setVolume(level)`

**現在の `index.html` はこれらを読み込んでない**。`player.js` / `enemy.js` には `typeof XxxVisual !== 'undefined'` のガード付きフォールバック矩形描画が入ってるが、視覚的にはプレースホルダ状態。

## 今回の目的

ビジュアル部・サウンド部の成果物を取り込み、**完全に動作する完成版ゲーム** として `outbox/integrated/` 配下にまとめ直す。QA・公開部がこれを受け取って最終テスト＆公開準備に進む。

## 守備範囲（やること）

### 1. 統合ディレクトリ作成

`outbox/integrated/` を作り、以下のファイルを配置：

```
outbox/integrated/
  index.html           ← 統合版エントリポイント（visual + sound 読み込み込み）
  game.js              ← 必要に応じて修正済み
  player.js            ← ShipVisual を正しく呼ぶよう修正
  enemy.js             ← EnemyVisual を正しく呼ぶよう修正
  bullet.js            ← ObjectsVisual.drawBullet を呼ぶよう修正
  background.js        ← BackgroundVisual を使うよう修正
  collision.js         ← 変更不要の見込み
  ship_visual.js       ← ビジュアル部からコピー
  enemy_visual.js      ← ビジュアル部からコピー
  background_visual.js ← ビジュアル部からコピー
  objects_visual.js    ← ビジュアル部からコピー
  effects_visual.js    ← ビジュアル部からコピー
  sound.js             ← サウンド部からコピー
```

### 2. 座標系の整合

- `player.js` は左上座標 `(x, y)` + `width/height` で管理。`ShipVisual.draw` は **中心座標** を要求。
  → `drawShip(ctx, x + width/2, y + height/2, width, height, frameCount)` のように変換して呼ぶ。
- `enemy.js`・`bullet.js` も同様に中心座標変換すること。
- `frame` パラメータは `game.js` のフレームカウンタを受け渡すか、各クラス内でインクリメント。

### 3. サウンド統合

- `game.js` 内で `const sound = new SoundManager();` を生成。
- **初回ユーザー操作**（キーダウンやクリック）で `sound.resume()` を呼ぶ（ブラウザ自動再生ポリシー対応）。
- タイトル → プレイ遷移で `sound.playBGM()`、クリア/ゲームオーバー遷移で `sound.stopBGM()` + `sound.playSE('clear' | 'gameover')`。
- 弾発射時 `sound.playSE('shoot')`、敵撃破時 `sound.playSE('explosion')`、自機被弾時 `sound.playSE('playerHit')`。

### 4. 背景

- `BackgroundVisual` の API（`initStars` / `updateStars` / `drawStars` もしくは類似名）をファイル冒頭コメントで確認し、ゲームループに組み込む。
- 既存 `background.js` は `BackgroundVisual` を呼ぶ薄いラッパーにするか、`BackgroundVisual` 直接利用で `background.js` を削除するか、どちらでも可。シンプルな方を選ぶ。

### 5. ゴール描画・エフェクト

- 30秒経過後のゴール出現時、`ObjectsVisual.drawGoal(ctx, x, y, frame)` で描画。
- 敵破壊時の爆発エフェクトは `effects_visual.js` の API を読んで適用（無理なら省略して報告書に記載）。

### 6. `index.html` の script タグ順序

### 7. 動作確認（静的チェック）

ローカルブラウザ起動は環境都合でできなくてもOKだが、**コード上で以下を検証**：

- 全 `*.js` が構文エラー無し（`node --check` もしくは目視チェック）
- `game.js` のゲームループが `BackgroundVisual` / `ShipVisual` / `EnemyVisual` / `ObjectsVisual` / `SoundManager` を正しく呼んでる
- `typeof XxxVisual !== 'undefined'` フォールバックは残しても良いが、統合版では **primary path が Visual モジュールを使うこと**
- タイトル状態・プレイ状態・クリア状態・ゲームオーバー状態のすべてで描画呼び出しが揃ってる
