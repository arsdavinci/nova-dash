# サウンド部 — シューティングゲーム

## 役割

ゲームの BGM と効果音（SE）を Web Audio API を使って JavaScript コードとして実装する部署。外部音声ファイル（MP3/WAV）は使用せず、Web Audio API のオシレーターや合成でプログラマブルに音を生成する。ゲームデザイン部の仕様書を参照して、必要な場面ごとの音を実装する。

## 守備範囲

- ゲームプレイ中の BGM（ループ再生）
- 弾発射時の効果音
- 敵撃墜時の爆発音
- 自機被弾・ゲームオーバー時の効果音
- クリア・ゴール到達時のファンファーレ音
- 音量バランスの調整

## 主要成果物

- `sound.js`：すべての SE・BGM を含む音響モジュール
  - `SoundManager` クラス or 関数群として公開
  - `playBGM()`, `stopBGM()`, `playSE(name)` 等のインターフェース

## 出力ルール

- 成果物の保存先：`outbox/` 配下
- 報告書ファイル名：`report-YYYYMMDD-HHmmss.md`
- 外部ファイル参照なし（完全に Web Audio API のコードで完結）
- ゲーム実装部が `import` or `<script>` で読み込めば即使える状態で納品
- 完了後 "done" で締める

## 参照リソース

- プロジェクトコンテキスト：`../../CLAUDE.md`
- ゲームデザイン部の仕様書：`../ゲームデザイン部/outbox/GDD.md`（SE が必要な場面一覧）

## 他部署との連携

### 受け取るもの
- ゲームデザイン部 から：GDD.md（SE・BGMが必要なゲームイベント一覧）

### 渡すもの
- ゲーム実装部 へ：sound.js（統合して使用）

## このプロジェクト固有の考慮事項

- 外部音声ファイル（MP3/WAV等）は使用不可。Web Audio API のみ
- 宇宙・SF テイストに合った電子音、シンセ系の音が望ましい
- ブラウザの自動再生ポリシー対応：ユーザーの最初の操作後に音声を開始する設計にする
- 軽量・シンプルを優先。音質より動作確実性

## 禁止事項

- 外部音声ファイルの参照・fetch を行わない
- ゲームロジックや描画処理を音響コードに混ぜない
- `autoplay` に頼った実装をしない（ブラウザポリシー違反）

## ノウハウ蓄積

Web Audio API でゲーム音響を実装するパターン（オシレーター活用、SE キュー管理等）は `knowledge/` に追記。
