## 作業内容

### 公開インターフェース

```javascript
class SoundManager {
  constructor()                // AudioContext 生成（suspended）、マスターゲイン用意
  resume()                     // ユーザー操作時に AudioContext を resume
  playBGM()                    // ループ再生開始
  stopBGM()                    // 停止
  playSE(name)                 // 'shoot' | 'explosion' | 'playerHit' | 'gameover' | 'clear'
  setVolume(level)             // 0.0〜1.0
}
```

### 各音の実装

仕様書 §2-1 〜 §2-6 の数値（波形・周波数・ADSR・持続時間）を **そのまま守って実装**：

- **shoot**: square, 1200Hz→800Hz, 40ms
- **explosion**: white noise + lowpass(1000Hz), ADSR 5/100/0.3/300ms
- **playerHit**: sawtooth, 400Hz→150Hz, 250ms
- **gameover**: sine 3音アルペジオ（440→330→220Hz）、各350ms
- **clear**: square 4音上昇（523→659→784→1047Hz）、各250ms
- **BGM**: 仕様書 §3 準拠。Cマイナー、BPM120、8小節=16秒ループ。Bass + Lead + Pad + Rhythm の4レイヤー

### BGM の実装方針

仕様書 §3 の4レイヤー構成を **Web Audio API のスケジューリング** で実現：

- 各レイヤーのノート列を `audioContext.currentTime` 基準でスケジュール
- 16秒経過後に次のループ分を再スケジューリング（`setTimeout` or `onended` で再帰）
- `stopBGM()` で全ノードを stop + disconnect

重すぎず、でも「宇宙SF」の雰囲気が出ること優先。**音質より動作確実性**（CLAUDE.md 指示）。

## エクスポート形式

ゲーム実装部は `<script>` タグで読み込むので：

```javascript
// ファイル末尾
if (typeof module !== "undefined" && module.exports) {
  module.exports = SoundManager;
}
if (typeof window !== "undefined") {
  window.SoundManager = SoundManager;
}
```

ゲーム実装部は `new window.SoundManager()` で使用、または `typeof SoundManager !== "undefined"` でガード。

## 自動再生ポリシー対応（必須）

- `constructor()` で AudioContext 生成だけ。**音は出さない**
- `resume()` 経由でユーザー操作時に `audioContext.resume()` を呼ぶ
- `Safari` 対応：`window.AudioContext || window.webkitAudioContext`
