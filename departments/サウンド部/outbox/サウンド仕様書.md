# サウンド仕様書 — 宇宙シューティング（SPACE SHUTTLE）

バージョン: 1.0  
作成: サウンド部  
参照元: GDD.md（ゲームデザイン部）  
対象: ゲーム実装部（sound.js を組み込む側）

---

## 1. サウンド要素一覧

| SE名 | トリガー条件 | 音の性質 |
|---|---|---|
| BGM | ゲームプレイ開始（Space/Enter押下後）〜ゲームオーバーまたはクリアまで | ループ、SF電子音。テンポは中程度（BPM=120）。緊張感のある宇宙系シンセ |
| shoot | Aキー押下（発射クールダウン10frame制御はゲーム側） | 短い高周波のピュンという発射音（50ms以下） |
| explosion | 敵（UFO/隕石）の HP が 0 になった瞬間 | ノイズ系の爆発音、0.3〜0.5秒 |
| playerHit | 自機被弾（HP減少時）。無敵時間中は発火しない | 低めのダメージ音、不快すぎない程度。0.2〜0.3秒 |
| gameover | HP=0 でゲームオーバー確定時 | 下降する電子音、重厚感あり。0.8〜1.5秒 |
| clear | 自機がゴール（宇宙ステーション）に接触した瞬間 | 上昇するファンファーレ。明るく短め（1〜2秒） |

## 2. 各サウンドの実装方針

### 2-1. BGM

GDDの状態遷移より、BGMはゲームプレイ中（状態: `playing`）のみ再生。タイトル・ゲームオーバー・クリア画面では停止。

- **波形**: 複数オシレーター（square + sawtooth + triangle の組み合わせ）
- **実装アプローチ**: Web Audio API の AudioBufferSourceNode を使ったオフラインレンダリング or リアルタイムスケジューリングによるループ生成
- **エンベロープ**: BGMは持続音なのでADSRより音量自体のゲイン調整を使用
- **詳細**: 後述「3. BGM設計」を参照

### 2-2. shoot（発射音）

- **波形**: square wave（矩形波）
- **周波数**: 1200 Hz → 800 Hz（短時間で下降）
- **持続時間**: 0.04秒（40ms）
- **エンベロープ**:
  - Attack: 0秒（即起動）
  - Decay: 0.04秒で 0 へ
  - Sustain: なし
  - Release: なし（Decay で完結）
- **実装アプローチ**: OscillatorNode + GainNode。frequency.linearRampToValueAtTime で高→低変化。gainNode.gain を 0.04秒で 0 にフェードアウト。

```
Oscillator(square, 1200Hz)
  → frequency.linearRampToValueAtTime(800, t+0.04)
  → GainNode(0.3 → 0 over 0.04s)
  → destination
```

### 2-3. explosion（爆発音）

- **波形**: AudioBuffer にホワイトノイズを生成 + BiquadFilterNode でローパスフィルタ
- **周波数**: ノイズ全帯域 → ローパスで 1000 Hz 以下を残す
- **持続時間**: 0.4秒
- **エンベロープ**:
  - Attack: 0.005秒
  - Decay: 0.1秒（ピーク → 0.3）
  - Sustain: 0.3（ゲイン）
  - Release: 0.3秒で 0 へフェードアウト
- **実装アプローチ**: AudioContext.createBuffer でホワイトノイズバッファを生成し AudioBufferSourceNode で再生。BiquadFilter（lowpass, frequency=1000）を通してモコッとした爆発感を演出。

```
BufferSource(WhiteNoise)
  → BiquadFilter(lowpass, 1000Hz)
  → GainNode（ADSR: 0.005/0.1/0.3/0.3）
  → destination
```

### 2-4. playerHit（自機被弾音）

- **波形**: sawtooth wave（ノコギリ波）
- **周波数**: 400 Hz → 150 Hz（下降）
- **持続時間**: 0.25秒
- **エンベロープ**:
  - Attack: 0.005秒
  - Decay: 0.05秒
  - Sustain: 0.4（ゲイン）
  - Release: 0.2秒で 0 へ
- **実装アプローチ**: OscillatorNode（sawtooth）+ GainNode。frequency を 400→150 Hz に 0.25秒かけてramp。

```
Oscillator(sawtooth, 400Hz → 150Hz over 0.25s)
  → GainNode（ADSR適用）
  → destination
```

### 2-5. gameover（ゲームオーバー音）

- **波形**: sine wave（正弦波）複数音を時間差で鳴らす
- **周波数**: 440 Hz → 330 Hz → 220 Hz（3音の下降アルペジオ）
- **持続時間**: 各音0.35秒、合計 1.0秒
- **エンベロープ（各音）**:
  - Attack: 0.01秒
  - Decay: 0.1秒
  - Sustain: 0.6（ゲイン）
  - Release: 0.2秒で 0
- **実装アプローチ**: 3つの OscillatorNode をそれぞれ time=t, t+0.3, t+0.6 でスタートしてスケジューリング。

```
Oscillator(sine, 440Hz) @ t+0.0
Oscillator(sine, 330Hz) @ t+0.3
Oscillator(sine, 220Hz) @ t+0.6
各 → GainNode（ADSR）→ destination
```

### 2-6. clear（クリアファンファーレ）

- **波形**: square wave（矩形波）または triangle wave（三角波）。明るさ重視で square を使用
- **音列**: ド（523Hz）→ ミ（659Hz）→ ソ（784Hz）→ 高ド（1047Hz）の上昇アルペジオ
- **持続時間**: 各音0.25秒、合計 1.0秒（最終音は0.4秒で余韻）
- **エンベロープ（各音）**:
  - Attack: 0.01秒
  - Decay: 0.05秒
  - Sustain: 0.7（ゲイン）
  - Release: 0.15秒で 0（最終音のみ0.3秒）
- **実装アプローチ**: 4つの OscillatorNode を順次スケジューリング。BGMがあれば同時に stopBGM() を呼ぶ。

```
Oscillator(square, 523Hz) @ t+0.0
Oscillator(square, 659Hz) @ t+0.25
Oscillator(square, 784Hz) @ t+0.5
Oscillator(square, 1047Hz) @ t+0.75
各 → GainNode（ADSR）→ destination
```

## 3. BGM 設計

### 3-1. テンポ・リズム構造

- **BPM**: 120（1拍 = 0.5秒、1小節 = 2秒）
- **拍子**: 4/4拍子
- **構成**: 8小節（16秒）を1ループ単位とする
- **ループ方法**: Web Audio API のスケジューリングで AudioBufferSourceNode を loop=true で再生。あるいは 16秒分の AudioBuffer をオフラインレンダリングしてループ再生。

### 3-2. 使用スケール・和音

- **スケール**: Cマイナースケール（C, D, Eb, F, G, Ab, Bb）
  - SF・宇宙のダーク感にマッチ
- **主和音進行**: Cm → Ab → Bb → Cm（単純な4コード繰り返し）
  - Cm: C(130Hz), Eb(156Hz), G(196Hz)
  - Ab: Ab(104Hz), C(130Hz), Eb(156Hz)
  - Bb: Bb(117Hz), D(147Hz), F(175Hz)

### 3-3. レイヤー構成

#### ベースライン（Bass）
- **波形**: sawtooth（ノコギリ波）
- **周波数帯**: 65〜130 Hz（低音域）
- **音量ゲイン**: 0.15
- **リズム**: 和音のルート音を4分音符で刻む
- **具体音列（8小節=16秒）**: C(65Hz)×4拍 → Ab(52Hz)×4拍 → Bb(58Hz)×4拍 → Cm和音ルートに戻る

#### メロディライン（Lead）
- **波形**: square wave（矩形波）+ OscillatorNode
- **周波数帯**: 260〜520 Hz（中音域）
- **音量ゲイン**: 0.12
- **リズム**: 8分音符〜16分音符の混合。シンプルな反復フレーズ
- **音列の方針**: Cマイナースケールから5〜7音を使ったモチーフを2小節で作り、4回繰り返す

#### パッドサウンド（Pad）
- **波形**: sine wave（正弦波）
- **周波数帯**: 130〜260 Hz（中低音域）
- **音量ゲイン**: 0.08（控えめに）
- **役割**: 和音の厚みを増す背景音。LFO（低周波オシレーター）でビブラートをかけて電子感を演出
- **LFO**: frequency=4Hz、depthは±2Hz

#### パーカッション（Rhythm）
- **音源**: ノイズバッファをフィルタ加工（バスドラ風・スネア風）
- **音量ゲイン**: 0.1
- **パターン**: 4/4で 1・3拍にバスドラ（lowpass noise, cutoff=200Hz）、2・4拍にスネア（bandpass noise, center=2000Hz）

### 3-4. 全体音量バランス

| レイヤー | ゲイン |
|---|---|
| Bass | 0.15 |
| Lead | 0.12 |
| Pad | 0.08 |
| Rhythm | 0.10 |
| マスターゲイン | 0.5（setVolume で変更可） |

## 4. SoundManager インターフェース設計

### 4-1. クラス設計

```javascript
class SoundManager {
  constructor()
  // AudioContext を生成（suspended 状態で起動）
  // マスターゲインノード（GainNode）を生成し destination に接続
  // 内部状態: _ctx, _masterGain, _bgmNodes[], _isBgmPlaying

  playBGM()
  // BGM を開始（ループ再生）
  // 内部で _startBGM() を呼び、_isBgmPlaying を true にする
  // 既に再生中なら何もしない

  stopBGM()
  // BGM を停止・後処理（ノードの disconnect/stop）
  // _isBgmPlaying を false にする

  playSE(name)
  // name: 'shoot' | 'explosion' | 'playerHit' | 'gameover' | 'clear'
  // 対応する内部メソッド（_playShoot 等）を呼ぶ
  // 不明な name は console.warn して無視

  setVolume(level)
  // level: 0.0〜1.0
  // _masterGain.gain.setValueAtTime(level, _ctx.currentTime) で反映

  resume()
  // AudioContext が suspended の場合に resume() を呼ぶ
  // ユーザー操作ハンドラ（click/keydown）から呼ばれる想定
}
```

### 4-2. 内部メソッド（非公開）

```javascript
// BGM 関連
_startBGM()       // BGMレイヤーの AudioNode を組み立て、スケジューリング開始
_stopBGMNodes()   // _bgmNodes 内の全ノードを stop() して配列クリア

// SE 関連（各1〜3個の AudioNode を生成してすぐ再生、再生後は自動 GC）
_playShoot()
_playExplosion()
_playPlayerHit()
_playGameover()
_playClear()

// ユーティリティ
_createNoiseBuffer(duration)   // ホワイトノイズの AudioBuffer を生成して返す
_scheduleNote(freq, wave, start, duration, gain)  // 単音スケジューリングのヘルパー
```

### 4-3. 使用例（ゲーム実装部向け）

```javascript
// 初期化（ゲーム開始前、1度だけ）
const sound = new SoundManager();

// ユーザー操作時（最初の Space/Enter キー押下）
document.addEventListener('keydown', () => {
  sound.resume();
}, { once: true });

// ゲームプレイ開始時
sound.playBGM();

// 弾を発射した瞬間
sound.playSE('shoot');

// 敵が撃墜された瞬間
sound.playSE('explosion');

// 自機が被弾した瞬間（HP減少時のみ、無敵判定はゲーム側で管理）
sound.playSE('playerHit');

// ゲームオーバー確定時
sound.stopBGM();
sound.playSE('gameover');

// クリア確定時
sound.stopBGM();
sound.playSE('clear');

// 音量変更（0.0〜1.0）
sound.setVolume(0.7);
```

## 5. ブラウザ自動再生ポリシー対策

### 5-1. 問題

Chrome・Firefox・Safari などの主要ブラウザは、**ユーザー操作（クリック・キー押下）なしに AudioContext を resume することを禁止** している。

`new AudioContext()` 直後は `state === 'suspended'` となり、音が鳴らない。

### 5-2. 対応パターン

```javascript
class SoundManager {
  constructor() {
    this._ctx = new (window.AudioContext || window.webkitAudioContext)();
    // この時点では suspended。まだ音は出さない
  }

  resume() {
    if (this._ctx.state === 'suspended') {
      this._ctx.resume();
    }
  }
}
```

ゲーム側（実装部）では、タイトル画面で Space/Enter を受け取った瞬間に `sound.resume()` を呼ぶ。その後 `sound.playBGM()` を呼べば正常に再生される。

```javascript
// ゲーム実装部側の想定コード（参考）
function handleTitleInput(key) {
  if (key === ' ' || key === 'Enter') {
    sound.resume();   // AudioContext を resume
    sound.playBGM();  // BGM 開始
    startGame();      // ゲームロジック開始
  }
}
```

### 5-3. モバイル対応（オプション）

タッチイベントも対象とする場合、`touchstart` イベントでも `sound.resume()` を呼ぶ。GDDではモバイル対応について明記がないため、オプション扱い。

### 5-4. Safari（webkitAudioContext）

```javascript
const AudioContextClass = window.AudioContext || window.webkitAudioContext;
this._ctx = new AudioContextClass();
```

上記のフォールバックで Safari にも対応する。

## 6. 実装優先順位

| 優先度 | サウンド | 理由 |
|---|---|---|
| 1（必須） | shoot | ゲームの基本ループ（弾発射）に直結。最頻出SE |
| 1（必須） | explosion | 敵撃墜のフィードバック。ゲームの手応えに重要 |
| 2（必須） | playerHit | 被弾フィードバック。HP管理と連動 |
| 2（必須） | gameover | ゲームオーバー状態の確認に必要 |
| 3（推奨） | clear | クリア演出として重要だが、クリア確認は画面表示で代替可能 |
| 4（推奨） | BGM | 雰囲気の向上に大きく貢献。ただしSEのみでも動作する |

### 実装ステップ推奨順

1. `SoundManager` クラスの骨格を作り、`resume()` + `playSE()` のディスパッチを実装
2. `_playShoot()` を実装して動作確認
3. `_playExplosion()`, `_playPlayerHit()`, `_playGameover()` を実装
4. `_playClear()` を実装
5. `_startBGM()` を実装（最後。複雑なため）
6. 音量バランス調整（`setVolume()` 経由）

## 7. ファイル構成

sound.js は単一ファイルで完結する。

```
projects/shooting-game/
└── departments/
    └── サウンド部/
        └── outbox/
            └── sound.js    ← SoundManager クラスをエクスポート（またはグローバル変数）
```

ゲーム実装部は `<script src="sound.js"></script>` で読み込み、`new SoundManager()` で使用する。

モジュール形式（`export default SoundManager`）と非モジュール形式（グローバル変数 `window.SoundManager`）の両対応を推奨。詳細はゲーム実装部と調整する。
