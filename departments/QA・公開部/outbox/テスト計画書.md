バージョン: 1.0
作成: QA・公開部
参照元: GDD.md v1.0（ゲームデザイン部）

---

## 1. テスト方針

コード上で仕様との照合を行うスタティックレビュー方式をベースとする。
実装されたコードを GDD の数値・ロジック定義と突き合わせ、差異をバグとして記録する。
クロスブラウザ確認はブラウザ依存 API の使用箇所をコードレビューで代替する。

## 3. バグ報告フォーマット

バグ発見時は以下のテンプレートに従って `バグレポート.md` へ追記する。

```markdown
### BUG-<連番>

| 項目 | 内容 |
|---|---|
| タイトル | <バグの概要（1行）> |
| 関連テストID | <T-XXX> |
| 再現手順 | 1. ... / 2. ... / 3. ... |
| 期待動作 | <仕様上あるべき動作> |
| 実際の動作 | <実際に起きていること> |
| 深刻度 | Critical / High / Medium / Low |
| 対応状況 | 未対応 / 修正中 / 修正済み |
| 備考 | <コードの該当箇所、原因推定等> |
```

### 深刻度定義

| 深刻度 | 定義 |
|---|---|
| Critical | ゲームが起動しない、クリア不可能、ゲームオーバーにならない等の致命的不具合 |
| High | 必須操作（移動・射撃・衝突判定）が機能しない |
| Medium | 仕様と数値が異なる・HUDが正しく表示されない等 |
| Low | 視覚的なずれ・推奨機能の欠如 |

## 4. テスト合否判定基準（公開 OK の定義）

### 必須条件（すべて満たすこと）

以下の条件をすべて満たした場合のみ「公開 OK」と判定する。

1. **Critical・High バグがゼロであること**
2. **必須（優先度: 必須）テスト項目をすべてパスしていること**
   - T-001〜T-011（操作系）
   - T-013〜T-025（弾・ゲームロジック）
   - T-026〜T-036（敵・スクロール）
   - T-038〜T-048（タイマー・ゴール・クリア）
   - T-049〜T-053（ゲームオーバー・リトライ）
   - T-057〜T-060（パフォーマンス・クロスブラウザ）
3. **ゲームが 30 秒以内にクリア可能であること（ゲームバランス確認）**
   - GDD タイムラインと敵出現パターンを照合し、達成可能性を確認

### 許容条件（あっても公開可）

- Medium・Low の未修正バグ（既知不具合として README に明記する）
- 推奨・オプション機能の欠如

### 判定フロー

```
必須テストのコードレビュー照合
   ↓ 全パス？
   Yes → Critical/High バグ 0 件確認
            ↓ 0 件？
            Yes → ゲームバランス確認（30秒クリア可能か）
                     ↓ OK？
                     Yes → 公開 OK
                     No  → ゲームデザイン部に差し戻し
            No  → ゲーム実装部にバグ修正依頼
   No  → ゲーム実装部に修正依頼
```
