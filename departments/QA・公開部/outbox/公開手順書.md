# 公開手順書 — 宇宙シューティング（SPACE SHUTTLE）

バージョン: 1.0
作成: QA・公開部

---

## 概要

このゲームは HTML5 Canvas + JavaScript による静的ファイル構成のため、サーバーサイド処理は不要。
GitHub Pages（無料・推奨）または Netlify（無料・簡単）でホスティングできる。

---

## 1. 公開ファイルの構成

公開に必要なファイル構成は以下の通り。

```
shooting-game/          ← リポジトリのルート（またはサブディレクトリ）
├── index.html          ← ゲームのエントリーポイント
├── game.js             ← ゲームロジック本体
├── README.md           ← プレイヤー向け説明（本書末尾のテンプレートを使用）
└── assets/             ← 存在する場合のみ
    ├── sounds/         ← BGM・SE ファイル（Web Audio API で生成する場合は不要）
    └── images/         ← 画像素材（Canvas 描画のみの場合は不要）
```

**重要:** `index.html` がリポジトリルートまたは公開ルートに存在することを確認する。
ゲーム実装部の成果物がどのファイル名・構成になっているかを受領時に確認すること。

## 2. GitHub Pages での公開手順

### Step 1: GitHub アカウントの準備

1. [https://github.com](https://github.com) にアクセスしてログイン（アカウントがなければ無料登録）

### Step 2: リポジトリの作成

1. GitHub の右上の「+」ボタン→「New repository」をクリック
2. 以下の設定でリポジトリを作成する：
   - **Repository name:** `shooting-game`（任意の名前でOK）
   - **Visibility:** Public（GitHub Pages の無料枠は Public 必須）
   - **Initialize with README:** チェックしない（後でファイルをアップロードするため）
3. 「Create repository」をクリック

### Step 3: ゲームファイルのアップロード

**方法A: GitHub Web UI でアップロード（Git 不要・初心者向け）**

1. 作成したリポジトリのページで「uploading an existing file」リンクをクリック
2. `index.html`、`game.js`、`README.md`（および `assets/` フォルダ）をドラッグ&ドロップ
3. 「Commit changes」ボタンをクリック

**方法B: Git コマンドでアップロード（推奨）**

```bash
# ローカルでゲームファイルがあるディレクトリに移動
cd /path/to/shooting-game

# Gitリポジトリを初期化
git init

# ファイルをステージング
git add index.html game.js README.md
# assets/ があれば: git add assets/

# コミット
git commit -m "Initial commit: SPACE SHUTTLE game"

# GitHub リポジトリをリモートとして追加（URLは自分のリポジトリに合わせる）
git remote add origin https://github.com/<ユーザー名>/shooting-game.git

# プッシュ
git branch -M main
git push -u origin main
```

### Step 4: GitHub Pages の有効化

1. リポジトリページの「Settings」タブをクリック
2. 左サイドバーの「Pages」をクリック
3. 「Source」セクションで以下を設定：
   - **Branch:** `main`
   - **Folder:** `/ (root)`
4. 「Save」ボタンをクリック

### Step 5: 公開URLの確認

1. 設定保存後、同じ Pages 設定ページに戻ると数分後に以下のURLが表示される：
   ```
   https://<ユーザー名>.github.io/shooting-game/
   ```
2. URLをクリックしてゲームが表示されることを確認する
3. **初回デプロイは反映まで 1〜3 分かかることがある**

### Step 6: ファイル更新時の手順

```bash
# ファイルを編集後
git add <変更したファイル>
git commit -m "Fix: <修正内容の説明>"
git push origin main
```

GitHub Pages は push 後 数分で自動更新される。

## 3. Netlify での公開手順（オプション・ドラッグ&ドロップ）

GitHub アカウントなしでも公開できる最速の方法。

### Step 1: Netlify Drop へアクセス

1. ブラウザで [https://app.netlify.com/drop](https://app.netlify.com/drop) を開く
   （Netlify アカウントは任意。なしでも一時公開できる）

### Step 2: ゲームフォルダをドロップ

1. `shooting-game/` フォルダ全体をブラウザの Drop ゾーンにドラッグ&ドロップする
   - フォルダ内に `index.html` が含まれていることを確認すること

### Step 3: 公開URLの取得

1. アップロードが完了すると、ランダムなサブドメインの URL が発行される：
   ```
   https://random-name-12345.netlify.app
   ```
2. URLをクリックしてゲームが表示されることを確認する

### Step 4: URL の固定化（オプション）

1. Netlify に無料アカウントを作成してログイン
2. サイト設定で「Site name」を変更することで URL をカスタマイズできる：
   ```
   https://space-shuttle-game.netlify.app
   ```

### Step 5: 継続更新の設定（オプション）

GitHub リポジトリと Netlify を連携すると、push するたびに自動デプロイされる：

1. Netlify の「Site settings」→「Build & deploy」→「Link repository」
2. GitHub のリポジトリを選択
3. Build settings はデフォルトのまま（静的サイトのためビルド不要）

## 5. プレイヤー向け README テンプレート

以下を `README.md` としてリポジトリルートに配置する。
`<プレイURL>` と `<ユーザー名>` は実際の値に置き換えること。

---

```markdown
# SPACE SHUTTLE — 宇宙シューティングゲーム

宇宙空間を舞台にした縦スクロールシューティングゲームです。
スペースシャトルを操り、UFO や隕石を撃墜しながら宇宙ステーションへの到達を目指しましょう。
ゲーム開始から約30秒で宇宙ステーションが出現します。シンプルな操作でブラウザからすぐに遊べます。

## プレイ

**[今すぐプレイ](<プレイURL>)**

## 操作方法

| キー | 操作 |
|---|---|
| 矢印キー（↑↓←→） | 自機（スペースシャトル）の移動 |
| A キー | 弾の発射（押しっぱなしで連射） |
| Space / Enter | ゲーム開始（タイトル画面から） |
| R | リトライ（クリア・ゲームオーバー画面から） |

## クリア条件

ゲーム開始から約30秒後に画面上部から宇宙ステーションが出現します。
自機を宇宙ステーションに接触させるとミッションクリアです。

## スコア

| イベント | 得点 |
|---|---|
| UFO 撃墜 | +100 点 |
| 隕石 撃墜 | +300 点 |
| クリアボーナス | +1,000 点 |
| 残 HP ボーナス | +200 点 × 残 HP 数 |
```

## 6. トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| GitHub Pages の URL にアクセスしても 404 になる | Settings→Pages の Branch 設定を確認。`main` ブランチが選択されているか確認。反映に数分かかることがある |
| ゲームが表示されるが操作できない | `index.html` と `game.js` が同じディレクトリにあるか確認。`<script src="game.js">` のパスが正しいか確認 |
| BGM・SE が鳴らない | ブラウザのオートプレイ制限により、ユーザー操作後でないと音声が再生されない仕様。Space/Enter でゲームを開始する手順で再生される実装になっているか確認 |
| Netlify Drop でアップロードしたが反映されない | フォルダ（index.html を含む）を直接ドロップしたか確認。zip 圧縮されたファイルはそのままでは使えない |
| スマートフォンで操作できない | このゲームはキーボード操作前提のため、スマートフォンへの対応は現バージョンのスコープ外。対応が必要であればタッチ操作の追加実装が必要 |
