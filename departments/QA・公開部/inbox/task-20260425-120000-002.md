実装部からの引き継ぎ事項：

- 音声はキーボード or クリック初回操作後に鳴り始める（自動再生ポリシー対応済み）

統合版に対する **コードレベル QA** を実施し、明らかな不具合があれば洗い出して報告。問題が無ければ（または軽微であれば）`outbox/dist/` に公開用ファイル一式を整備する。

実機ブラウザ起動はサブエージェント環境では不可なので、**静的解析中心** のテストを行う：

## 守備範囲（やること）

### 1. 静的コードレビュー

`integrated/` の全 JS / HTML を Read で確認し、以下をチェック：

- [ ] `index.html` の script 読み込み順が依存解決可能（前方参照無し）
- [ ] `window.ShipVisual` / `EnemyVisual` / `BackgroundVisual` / `ObjectsVisual` / `EffectsVisual` / `SoundManager` がそれぞれ「定義側ファイル」と「使用側ファイル」で整合
- [ ] `game.js` で `requestAnimationFrame` ループが回ってる
- [ ] キーイベントハンドラが矢印キー（ArrowUp/Down/Left/Right）と A キーを拾ってる
- [ ] タイトル/プレイ/クリア/ゲームオーバー の状態遷移が実装されてる
- [ ] 30秒タイマーまたは相当のフレーム数（30秒×60fps=1800フレーム前後）でゴール出現の処理がある
- [ ] 衝突判定が「弾vs敵」「自機vs敵」両方ある
- [ ] サウンド呼び出しが `shoot` / `explosion` / `playerHit` / `clear` / `gameover` の各シーンで揃ってる
- [ ] 自機初回操作前の `sound.resume()` 呼び出しが存在
- [ ] `console.log` などのデバッグ出力が残ってないか（残ってても重大ではないが報告）

### 2. テスト計画書との照合

`outbox/テスト計画書.md` に64項目あるが、全項目をブラウザで実施するのは無理なので、**コードを読めば判定できる項目だけピックアップ** して結果記録する。最低15項目はカバーすること。

### 3. バグ・不整合の検出

レビュー中に見つけた問題は **重大度別** に列挙：

- **致命的**：起動しない／ゲームループが止まる／ゴールに到達できない
- **重大**：主要機能が動かない（弾が出ない、敵が湧かない、衝突しない 等）
- **中**：表示崩れ／音が出ないシーンがある
- **軽微**：コメント不整合・未使用コード・気になる書き方

致命的・重大が見つかった場合は **修正案** をコード抜粋付きで提示。直接修正してもよい（ただし修正したファイルパスを明記）。

### 4. 公開用ディレクトリ整備

致命的・重大バグが無い（または修正済み）なら、`outbox/dist/` を作成して以下を配置：

```
outbox/dist/
  index.html         ← integrated からコピー
  *.js               ← integrated の全JSをコピー
  README.md          ← プレイ方法・操作・公開手順を記載した日本語README
```

`README.md` に書くこと：
- ゲームタイトル・概要
- 操作方法（矢印キー・Aキー・スペース/Enter・R）
- プレイ目標・クリア条件
- 起動方法（「`index.html` をブラウザで開くだけ」）
- 動作要件（モダンブラウザ Chrome / Edge / Firefox 推奨）
- 制作：Synapse Enterprise の表記

### 5. 公開手順書のチェック

既存の `outbox/公開手順書.md` を参照し、`dist/` をそのまま GitHub Pages / Netlify / Vercel 等に置けば公開できる状態になってるか最終確認。
