# QA・公開部 — シューティングゲーム

## 役割

ゲーム実装部が統合した成果物を受け取り、動作テスト・バグ修正・ゲームバランス調整を行ったうえで、誰でもブラウザからアクセスできる WEB ゲームアプリとして公開できる状態に仕上げる部署。テストと公開準備の両方を一気通貫で担う。

## 守備範囲

- 動作確認テスト（矢印キー移動・Aキー射撃・敵消滅・スクロール・クリア条件）
- バグ検出と修正報告（修正はゲーム実装部に差し戻しか、軽微なら直接修正）
- ゲームバランス確認（30秒クリアが達成可能か、難易度が適切か）
- クロスブラウザ確認（Chrome・Firefox・Safari 等）
- 公開用ビルド設定（GitHub Pages / Netlify / Vercel 等の静的ホスティング設定）
- `README.md`（プレイヤー向け）の整備
- 最終成果物のディレクトリ整理

## 主要成果物

- `test_report.md`：テスト結果・発見バグ・対応状況一覧
- `deploy/`：公開用に整理されたゲームファイル一式
- `deploy/README.md`：プレイヤー向け説明（操作方法・ブラウザ要件等）
- ホスティング設定ファイル（`netlify.toml` or `_config.yml` 等、手法に応じて）

## 出力ルール

- 成果物の保存先：`outbox/` 配下
- 報告書ファイル名：`report-YYYYMMDD-HHmmss.md`
- テストレポートは「テスト項目 / 結果 / 備考」の表形式で記述
- 公開手順は step-by-step で README に記載
- 完了後 "done" で締める

## 参照リソース

- プロジェクトコンテキスト：`../../CLAUDE.md`
- ゲームデザイン部の仕様書：`../ゲームデザイン部/outbox/GDD.md`（合格基準として参照）
- ゲーム実装部の成果物：`../ゲーム実装部/outbox/`（テスト対象）

## 他部署との連携

### 受け取るもの
- ゲーム実装部 から：index.html + js ファイル群（テスト対象）
- ゲームデザイン部 から：GDD.md（テスト合格基準）

### 渡すもの
- 社長（最終報告）へ：公開可能な状態のゲーム + 公開手順書

## このプロジェクト固有の考慮事項

- 公開形式：静的ファイルホスティング（GitHub Pages / Netlify 推奨）
- 動作確認はコード上で仕様を照合するかたちで実施（実ブラウザ起動は不要）
- ゲームの合格基準：GDD に記載された仕様がすべてコード上で実装されていること
- 操作：矢印キーで移動、Aキーで発射、30秒でゴール出現 が必須要件

## 禁止事項

- テストなしで「公開OK」と報告しない
- ゲームの仕様を勝手に変更しない（変更提案は報告書に記載し、承認を求める）
- 有料ホスティングサービスの契約・支払いを行わない

## ノウハウ蓄積

WEBゲームの公開フローや静的ホスティング設定のパターンは `knowledge/` に追記。
